

### ШАБЛОН: Подготовка к выполнению практического задания (Linux)

#### Этап 0: Проверка и Демонстрация (Начало экзамена)
1.  **Демонстрация интернета:** Открываю браузер (Firefox/Chrome) и захожу на `ya.ru` или `google.com` — показываю, что страница загружается.
2.  **Демонстрация ОС:** Открываю терминал (`Ctrl+Alt+T`) и ввожу:
    ```bash
    lsb_release -a
    ```
    (показываю, что стоит Ubuntu).

---

### ЧАСТЬ 1. Обязательные пункты (для «Удовлетворительно»)

*Эти пункты есть в каждом билете. Это ваш страховочный минимум.*

#### 1. Пользователи, группы и права (Аутентификация/Авторизация)
*   **Создание группы:**
    ```bash
    sudo groupadd developers
    ```
*   **Создание пользователей (например, `ivan` и `petr`):**
    ```bash
    sudo useradd -m -G developers ivan
    sudo passwd ivan  # Устанавливаю пароль (например, 123)
    sudo useradd -m -G developers petr
    sudo passwd petr
    ```
*   **Настройка прав доступа (пример: создаем папку проекта):**
    ```bash
    sudo mkdir /opt/project
    # Даем права группе developers на чтение/запись в эту папку
    sudo chown :developers /opt/project
    sudo chmod 770 /opt/project
    ```
*   **Журналы мониторинга (показываем, что логи пишутся):**
    ```bash
    # Смотрим логи авторизации
    sudo tail -f /var/log/auth.log
    # Или логи системы
    sudo journalctl -xe
    ```

#### 2. Резервное копирование, Образ системы, Точка восстановления
*   **Точка восстановления (создаем снепшот, если это виртуалка):**
    *Если в билете просят создать точку — мы работаем в VirtualBox, жмем `Машина -> Снимки -> Сделать снимок`.*
*   **Резервная копия (архивируем важную папку, например /etc):**
    ```bash
    sudo tar -czvf /tmp/backup_etc.tar.gz /etc
    ```
*   **Установочный образ системы (показываем, что мы скачали ISO или создаем образ диска с помощью `dd`):**
    ```bash
    # Если в виртуалке - показываем образ в настройках.
    # Для создания образа загрузочной флешки (имитация):
    sudo dd if=/dev/sda of=/tmp/disk_image.img bs=1M count=100
    ```

#### 3. Установка базового ПО
```bash
sudo apt update
sudo apt install nano vim gedit libreoffice gcc build-essential -y
```

#### 4. Установка виртуального принтера
```bash
sudo apt install cups-pdf -y
```
*После установки заходим в `Настройки -> Принтеры -> Добавить принтер` и выбираем `PDF`.*

### ЧАСТЬ 2. Основное задание из Билета (для «Хорошо»)

*Здесь нужно заменить текст в скобках `[...]` на то, что написано в вашем билете.*

**Ситуация:** Требуется установить ПО для [ВСТАВИТЬ ОПИСАНИЕ ИЗ БИЛЕТА, например: *разработки на Python/JS, СУБД, инструменты тестирования API*].

#### Шаг 1. Установка основного ПО
Обновляем репозитории:
```bash
sudo apt update
```

**Устанавливаем то, что просят в билете (список):**
*   *Пример 1 (Среда разработки):* Если просят Python/JS -> ставим VSCode или PyCharm.
*   *Пример 2 (СУБД):* Если просят базу данных -> ставим PostgreSQL или MySQL.
*   *Пример 3 (Тестирование API):* Если просят инструмент для тестов -> ставим Postman или Insomnia.

**Вот команды для самых частых программ (выбираете под билет):**
```bash
# Git (контроль версий)
sudo apt install git gitk -y   # gitk - это графический интерфейс

# СУБД MySQL
sudo apt install mysql-server -y

# СУБД PostgreSQL
sudo apt install postgresql postgresql-contrib -y

# Среда разработки VSCode (скачиваем .deb или через snap)
sudo snap install --classic code

# Инструмент для API (Postman)
sudo snap install postman

# Или curl/wget для проверки (если не графический интерфейс)
sudo apt install curl wget -y
```

#### Шаг 2. Базовая настройка и совместимость (Linux)
1.  **Запускаем службы (например, MySQL):**
    ```bash
    sudo systemctl start mysql
    sudo systemctl enable mysql
    sudo systemctl status mysql
    ```
2.  **Настройка совместимости (если в билете есть про "ограниченную цветовую палитру" и "масштабирование"):**
    *   Для приложений с высоким DPI:
        ```bash
        # Запускаем программу с отключением масштабирования (для старых приложений)
        QT_AUTO_SCREEN_SCALE_FACTOR=0 code
        ```
    *   Отключаем композицию рабочего стола (для устранения тормозов):
        *В настройках системы -> "Внешний вид" -> "Настройки окна" или через терминал для конкретного приложения:*
        ```bash
        # Пример для Wine-приложений
        export DISABLE_LAYER_AMD_SWITCHABLE_GRAPHICS=1
        ```
3.  **Настройка обмена данными (если нужно подключиться к серверу БД):**
    *   Открываем конфиг MySQL:
        ```bash
        sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf
        # Меняем bind-address = 0.0.0.0 (чтобы принимать подключения не только с localhost)
        ```

### ЧАСТЬ 3. Документация (для «Отлично»)

*Здесь вы оформляете документ прямо в терминале или в любом текстовом редакторе, показывая преподавателю готовый файл.*

#### 1. Создаем файл документации пользователя
```bash
nano ~/Руководство_пользователя.txt
```
**Структура документа (вставляем в файл и подставляем свою программу):**

```
=============================================================
       РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
       (Согласно ГОСТ Р 59795-2021)
=============================================================

1. Наименование программы: [Название программы, например, Postman API Client]

2. Краткое описание:
   Программа предназначена для [пишем, зачем она, например: тестирования REST API,
   отправки HTTP-запросов и анализа ответов сервера].

3. Запуск программы:
   - Через терминал: postman
   - Через графическое меню: Нажать на иконку Postman в меню приложений.

4. Запланированный программой выход (завершение):
   - Нажать кнопку "Закрыть" (крестик) в правом верхнем углу окна.
   - Или использовать комбинацию клавиш Ctrl+Q (если поддерживается).

5. Основные функции (кратко):
   - Создание запросов (GET, POST, PUT, DELETE).
   - Просмотр заголовков и тела ответа.
   - Сохранение запросов в коллекции.

=============================================================
```

#### 2. Создаем файл "Руководство по использованию"
```bash
nano ~/Руководство_по_использованию.txt
```
**Содержание:**

```
=============================================================
          РУКОВОДСТВО ПО ИСПОЛЬЗОВАНИЮ
=============================================================

1. Перечень выполняемых функций (для [Название программы]):
   - Функция 1: Отправка GET-запроса для получения данных.
   - Функция 2: Отправка POST-запроса с телом JSON для создания записей.
   - Функция 3: Импорт/Экспорт коллекций для обмена с командой.

2. Основные приемы работы:
   - Чтобы отправить запрос, нужно ввести URL в адресную строку,
     выбрать метод (GET/POST) и нажать кнопку "Send".
   - Для авторизации перейти на вкладку "Authorization" и выбрать тип (Bearer Token / Basic Auth).
   - Для сохранения результата нажать "Save Response" -> "Save to File".

3. Возможные проблемы и их решение:
   - Если сервер не отвечает: Проверить соединение (ping [адрес]).
   - Если ошибка 403: Проверить правильность токена авторизации.
=============================================================
```

### Ваш "План-шпаргалка" для действий на экзамене:

1.  **Включил комп** -> открыл браузер (показал интернет).
2.  **Открыл терминал** -> выполнил все команды из **Части 1** (пользователи, права, архивы, принтер).
3.  **Взял билет** -> посмотрел, какое ПО требуют (например, "СУБД" и "Среда разработки").
4.  **Выполнил команды из Части 2**, подставив нужные названия (apt install ...).
5.  **Создал** два текстовых файла (`nano ...`) и заполнил их текстом из **Части 3**, подставив название программы, которую только что установили.
6.  **Показал** преподавателю:
    *   Работающий терминал с логами (`auth.log`).
    *   Установленное ПО (например, открыл MySQL: `sudo mysql`).
    *   Содержимое созданных файлов документации (`cat ~/Руководство*`).

Этот шаблон покрывает 90% возможных заданий. Остальное (вроде "настройка ядра" или специфические утилиты) решается через `sudo apt install` нужной утилиты, если она есть в списке ресурсов.
